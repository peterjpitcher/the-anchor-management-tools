    - Import cn from '@/lib/utils' for class composition
    - Use @theme token utility classes ONLY (bg-primary, text-text, etc.) -- never hardcoded hex
    - Server components by default (no 'use client') EXCEPT Button and Modal which need interactivity
    - All components use forwardRef where they render a native HTML element
    - Named exports only (no default exports)

    **Button.tsx** ('use client'):
    - Props: variant ('primary'|'secondary'|'ghost'|'danger', default 'secondary'), size ('sm'|'md'|'lg', default 'md'), icon (string, optional), iconRight (string, optional), loading (boolean), plus ButtonHTMLAttributes
    - Variant styles:
      primary: bg-primary text-primary-fg border-primary hover:bg-primary-hover shadow inner glow
      secondary: bg-surface text-text border-border-strong hover:bg-surface-hover
      ghost: bg-transparent text-text border-transparent hover:bg-surface-hover
      danger: bg-danger text-white border-danger hover:brightness-95
    - Size styles use CSS var heights: sm=var(--spacing-btn-h-sm) px-2.5 text-xs, md=var(--spacing-btn-h) px-3 text-[13px], lg=var(--spacing-btn-h-lg) px-4 text-sm
    - Icon-only: when no children, button becomes square (w equals h)
    - Loading: shows spinner, sets opacity-50 cursor-not-allowed, disables button
    - Focus: focus-visible:outline-none focus-visible:shadow-ring
    - Active: active:translate-y-[0.5px]
    - Reference implementation is in 01-RESEARCH.md Pattern 2 -- follow it closely but import Icon from '@/ds/icons' (which may not exist yet -- use a simple SVG spinner inline for loading state, defer Icon integration to Plan 03 when icons are built)

    **Badge.tsx** (server component, no 'use client'):
    - Props: tone ('neutral'|'primary'|'success'|'warning'|'danger'|'info', default 'neutral'), dot (boolean, default false), children
    - Tone styles mapping -- each has bg, text, and border:
      neutral: bg-surface-2 text-text-muted border-border
      primary: bg-primary-soft text-primary-soft-fg border transparent
      success: bg-success-soft text-success-fg border transparent
      warning: bg-warning-soft text-warning-fg border transparent
      danger: bg-danger-soft text-danger-fg border transparent
      info: bg-info-soft text-info-fg border transparent
    - Base: inline-flex items-center gap-1.5 px-2 py-0.5 text-xs font-medium rounded-pill border
    - Dot: 6px circle with tone-matched background colour before text

    **Avatar.tsx** (server component):
    - Props: name (string), size ('sm'|'md'|'lg'|'xl', default 'md'), className
    - Sizes: sm=24px, md=32px, lg=40px, xl=56px with proportional text
    - Deterministic colour from name hash (pickColor): hash each char of name, mod 6 to pick from palette:
      ['#ef4444','#f97316','#eab308','#22c55e','#3b82f6','#8b5cf6']
    - Shows first letter of first name + first letter of last name (split on space), uppercased
    - Rounded-full, flex items-center justify-center, font-semibold, text-white
    - Also export AvatarStack: renders up to N avatars overlapping (-ml-2 after first), shows "+X" overflow badge

    **Alert.tsx** (server component):
    - Props: tone ('success'|'warning'|'danger'|'info'), title (string, optional), children (body), icon (ReactNode, optional)
    - Tone styles: border-l-4 with tone colour, bg uses tone-soft, text uses tone-fg
    - Layout: flex gap-3, icon on left (24px), title bold + body below

    **Modal.tsx** ('use client'):
    - Props: open (boolean), onClose (()=>void), title (string, optional), children (body), footer (ReactNode, optional), width ('sm'|'md'|'lg'|'xl', default 'md')
    - Uses Headless UI Dialog (import { Dialog, DialogBackdrop, DialogPanel, DialogTitle } from '@headlessui/react')
    - Backdrop: fixed inset-0 bg-black/50 with transition-opacity
    - Panel: bg-surface rounded-lg shadow-lg, centered with max-w based on width prop (sm=400px, md=500px, lg=640px, xl=800px)
    - Header: px-6 py-4 border-b border-border with DialogTitle
    - Body: px-6 py-4
    - Footer: px-6 py-4 border-t border-border flex justify-end gap-3
    - Transition: scale from 95% to 100%, opacity 0 to 1, duration 200ms

    **Skeleton.tsx** (server component):
    - Props: className, width (string), height (string), rounded ('sm'|'md'|'lg'|'full', default 'md')
    - Base: bg-surface-2 animate-pulse with rounded variant
    - Add @keyframes skel-shimmer to globals.css if not already present (background gradient sliding animation)

    **Empty.tsx** (server component):
    - Props: icon (ReactNode, optional), title (string), description (string, optional), action (ReactNode, optional)
    - Layout: flex flex-col items-center justify-center py-12 text-center
    - Icon: text-text-subtle mb-4, 48px size
    - Title: text-text font-semibold text-lg
    - Description: text-text-muted text-sm mt-1
    - Action: mt-4

    **Toast.tsx** ('use client'):
    - Props: tone ('success'|'warning'|'danger'|'info'|'neutral'), message (string), visible (boolean)
    - Layout: fixed bottom-4 right-4 z-50, bg-surface shadow-lg rounded-lg border-border px-4 py-3
    - Colour dot: 8px circle on left, colour matches tone
    - Auto-dismiss: useEffect with setTimeout (4 seconds default)
    - Slide-up animation on enter
    - Note: This is a standalone toast component. The existing react-hot-toast remains for legacy pages. Phase 2 migration will switch pages to this component.
