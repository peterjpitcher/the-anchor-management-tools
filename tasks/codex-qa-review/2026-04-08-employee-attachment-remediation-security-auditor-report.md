### SEC-001: Any authenticated user can bypass app RBAC and directly read/write/delete employee attachments
- **File:** [employeeActions.ts:756](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/app/actions/employeeActions.ts#L756), [employeeActions.ts:814](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/app/actions/employeeActions.ts#L814), [employeeActions.ts:854](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/app/actions/employeeActions.ts#L854), [client.ts:1](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/lib/supabase/client.ts#L1), [20251123120000_squashed.sql:4923](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/supabase/migrations/20251123120000_squashed.sql#L4923), [20251123120000_squashed.sql:4995](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/supabase/migrations/20251123120000_squashed.sql#L4995), [20260425100000_fix_employee_attachments_storage_bucket.sql:57](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/supabase/migrations/20260425100000_fix_employee_attachments_storage_bucket.sql#L57), [20260425100000_fix_employee_attachments_storage_bucket.sql:69](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/supabase/migrations/20260425100000_fix_employee_attachments_storage_bucket.sql#L69), [20260425100000_fix_employee_attachments_storage_bucket.sql:81](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/supabase/migrations/20260425100000_fix_employee_attachments_storage_bucket.sql#L81)
- **Severity:** Critical
- **Category:** Auth
- **OWASP:** A01 Broken Access Control
- **Description:** The reviewed server actions rely on `checkUserPermission()` as the enforcement point, but the underlying Supabase policies make that check non-binding. `employee_attachments` is open to all authenticated users with `USING (true) WITH CHECK (true)`, and `storage.objects` grants authenticated `INSERT`, `SELECT`, and `DELETE` for the entire `employee-attachments` bucket. Because the app also exposes a browser Supabase client, any logged-in user can call PostgREST/Storage directly from the browser and bypass `employees:view_documents`, `upload_documents`, and `delete_documents` entirely.
- **Impact:** Full exfiltration, tampering, and deletion of sensitive employee documents such as P60s and contracts by any authenticated user, not just users with document permissions.
- **Suggested fix:** Lock down both `employee_attachments` and `storage.objects` with RBAC-aware RLS, or make them service-role only and force all access through vetted server actions. Remove permissive `USING (true)` policies and restrict storage access by both bucket and expected path rules.

### SEC-002: File-type enforcement trusts attacker-controlled metadata, allowing dangerous file uploads
- **File:** [employees.ts:124](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/services/employees.ts#L124), [employees.ts:516](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/services/employees.ts#L516), [employeeActions.ts:789](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/app/actions/employeeActions.ts#L789), [AddEmployeeAttachmentForm.tsx:114](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/components/features/employees/AddEmployeeAttachmentForm.tsx#L114), [EmployeeAttachmentsList.tsx:172](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/components/features/employees/EmployeeAttachmentsList.tsx#L172)
- **Severity:** High
- **Category:** Input Validation
- **OWASP:** A05 Security Misconfiguration
- **Description:** Upload validation is based on `File.type` and does not verify file signatures or enforce safe extensions server-side. The original filename is preserved, the bytes are stored as supplied, and `maybeSendEmployeeAttachmentEmail()` re-sends the uploaded file using attacker-controlled filename/MIME metadata. A malicious user can upload a dangerous payload such as `.docm`, `.exe`, or HTML/polyglot content while setting an allowed MIME like `application/pdf` or `text/plain`, and the system will store it, offer it for view/download, and potentially email it from a trusted sender.
- **Impact:** Malware/phishing delivery through the employee document workflow, plus possible active-content execution when attachments are opened.
- **Suggested fix:** Validate magic bytes and extension server-side, normalize stored content type, reject executable or macro-enabled formats, add AV/quarantine scanning, and force download-only handling for anything not explicitly safe to render inline.
