## DST reality

The adapter comments say “Europe/London wall-clock,” but the implementation is `new Date(y, m - 1, d, hh, mm)`, which uses the browser/runtime local timezone, not an explicit London conversion: [adapters.ts](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/components/schedule-calendar/adapters.ts:12). On non-UK client timezones, the “London wall-clock” assumption is not guaranteed.

## Findings Summary

| ID | Severity | Scenario | What happens | Recommendation |
|---|---|---|---|---|
| WF-01 | High | Event booking counts after payment/lifecycle changes | `bookedSeatsCount` only counts `confirmed`; `pending_payment`, `visited_waiting_for_review`, `review_clicked`, and `completed` seats disappear from calendar/dashboard counts. | Align count statuses with the event booking lifecycle and runtime reservation semantics; centralize the status set. |
| WF-02 | High | Stripe/PayPal/API/SMS booking mutations | Several booking creation/confirmation/seat-increase paths mutate data without revalidating `/events` or dashboard, so counts and entries can stay stale. | Add shared post-booking revalidation for event detail, `/events`, dashboard tag, and affected list pages. |
| WF-03 | Medium | Cmd/Ctrl-click month/week entry | Anchors call `preventDefault()` and `router.push`, breaking native new-tab behavior for modified clicks. | Preserve native anchor behavior when `metaKey`, `ctrlKey`, `shiftKey`, or non-left click is present. |
| WF-04 | Medium | Overnight private booking in week/list | Week renders only in start-day column and can clip past visible range; list shows one start-date row with no `+1 day`. | Split overnight timed entries across days or explicitly render continuation indicators in week/list. |
| WF-05 | Medium | Past private bookings/parking on dashboard calendar | Past rows are fetched/capped but not passed to the redesigned calendar; no “showing last N” affordance exists. | Either pass past rows into the calendar or remove the unused fetch; add explicit capped-window messaging if caps remain. |
| WF-06 | Medium | Private booking past cap ordering | Past private bookings are limited after ascending ordering, which can select the earliest 50 in the 90-day window. | Query most recent 50 first, then reverse for chronological display. |
| WF-07 | Medium | Empty-day note creation | With permission, date-number buttons can open note creation on populated days and adjacent-month cells, not just empty in-month days. | Gate the button on `inMonth && dayEntries.length === 0`, or rename the behavior if notes are allowed on any day. |
| WF-08 | Medium | DST/client timezone | Calendar uses local browser timezone despite London-wall-clock comment and renders uniform 24-hour grids on DST days. | Use explicit London-aware date construction/formatting and decide whether DST days need non-uniform hour handling. |
| WF-09 | Low | Mobile first paint | Server and initial client render as desktop, then mobile switches to list after hydration. | Consider CSS-first responsive rendering or a mounted-state fallback to avoid mobile desktop flash. |
| WF-10 | Low | List Today anchor | `scrollIntoView` moves the parent shell/content pane, not a fixed list viewport; refresh remounts could jump back to Today. | Give the list a deliberate scroll container or make auto-anchor conditional/user-controlled after first mount. |
