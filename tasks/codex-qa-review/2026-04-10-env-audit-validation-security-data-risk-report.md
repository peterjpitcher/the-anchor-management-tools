6. **Finding 6: Environment variable exposure**
- `SEVERITY: HIGH`
- `1/2:` I found no `SUPABASE_SERVICE_ROLE_KEY` or `STRIPE_SECRET_KEY` usage in any `use client` file. The actual references are server-only. Evidence: [admin.ts](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/lib/supabase/admin.ts#L5), [import-messages.ts](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/app/actions/import-messages.ts#L110), [stripe.ts](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/lib/payments/stripe.ts#L98)

The API accepts that token as the full bearer credential. Evidence: [rota page](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/app/(authenticated)/rota/page.tsx#L114), [RotaFeedButton](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/app/(authenticated)/rota/RotaFeedButton.tsx#L7), [RotaFeedButton](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/app/(authenticated)/rota/RotaFeedButton.tsx#L111), [rota feed route](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/app/api/rota/feed/route.ts#L22), [rota feed route](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/app/api/rota/feed/route.ts#L48)
- `4:` I did not find any API route returning raw secret values in JSON/response bodies. Closest case is only naming a missing secret variable, not returning its value. Evidence: [stripe webhook](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/app/api/stripe/webhook/route.ts#L1177)
- `EXPLOITABILITY:` Any user who can access the rota page gets a reusable bearer URL for `/api/rota/feed`; they can copy/share it outside the app, and the feed endpoint does not require a user session once the token is known.
- `RECOMMENDATION:` Do not pass a global env-derived bearer token to the client. Replace it with per-user scoped feed tokens stored server-side, or proxy feed generation behind authenticated/session-bound checks. Rotate `ROTA_FEED_SECRET` after fixing this.
