**Spec defects**
- The spec’s TTL theory is not supported by repo defaults. SMS idempotency defaults to `336` hours / 14 days in [sms/safety.ts](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/lib/sms/safety.ts:196) and [.env.example](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/.env.example:21). Without a verified prod override or `idempotency_keys` failure, 15-minute reruns should hit `idempotency_conflict`, not resend.
- Fix 2 is too broad as written. `buildSmsDedupContext()` is global, so URL stripping would also change dedupe for other URL-bearing templates such as [table booking confirmation/payment](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/lib/table-bookings/bookings.ts:733), [event payment/manage](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/lib/events/event-payments.ts:433), [waitlist acceptance](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/app/g/%5Btoken%5D/waitlist-offer/confirm/route.ts:153), [parking payment request](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/lib/parking/payments.ts:181), and [private booking review](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/app/api/cron/private-booking-monitor/route.ts:902). That can suppress legitimate refreshed-link sends.
- Fix 1’s sample code is incomplete. With `.is('review_sms_sent_at', null)`, a no-row update is silent; checking only `sentAtError` does not prove the flag was written. The spec needs a read-back / `select()` path that distinguishes “already set” from “missing row / not written”.
- The event-booking version of Fix 1 must preserve `review_window_closes_at`. That field is consumed by [processReviewWindowCompletion()](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/app/api/cron/event-guest-engagement/route.ts:1293); if the split update does not persist it independently of the status change, lifecycle completion breaks.

The real migration defect is the stale squashed baseline at [20251123120000_squashed.sql](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/supabase/migrations/20251123120000_squashed.sql:8577).
- The spec misses the private-booking review path, which also creates a random `review_redirect` token before send and only marks `review_processed_at` afterward at [private-booking-monitor/route.ts](/Users/peterpitcher/Cursor/OJ-AnchorManagementTools/src/app/api/cron/private-booking-monitor/route.ts:852).
