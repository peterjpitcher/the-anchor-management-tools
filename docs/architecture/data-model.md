## Notes

- Names ending in `_view` or `_with_details` / `_with_costs` / `_with_prices` / `_stats` are **PostgreSQL views**, not base tables — they freeze their column list at creation time, so any underlying table change requires the view to be recreated (see `CLAUDE.md` → "Dropping columns or tables — mandatory function audit").
- `admin_users_view` joins auth users to staff metadata.
- `customer_messaging_health` is a derived view.
- `private_bookings_with_details` is the canonical read view for the private-bookings module.
- The `jobs` and `cron_job_runs` tables underpin the unified job queue triggered by `/api/jobs/process` every minute.
