## Shared Patterns

- All mutations call `revalidatePath('/relevant-path')` after success
- All mutations call `logAuditEvent({ user_id, operation_type, resource_type, operation_status })` via `actions/audit.ts`
- Auth client: `src/lib/supabase/server.ts` (cookie-based, respects RLS)
- Admin client: `src/lib/supabase/admin.ts` (service role, bypasses RLS — cron/system only)
