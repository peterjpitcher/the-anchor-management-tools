2. **Layout guard.** `src/app/(authenticated)/layout.tsx` re-verifies `auth.getUser()`, redirects to `/auth/login`, loads permissions via `getUserPermissions()`, and redirects portal-only users (zero management permissions or `portal_shift_manager` role) to `/portal/shifts`, with an admin-client fallback role check if the permissions RPC fails.
