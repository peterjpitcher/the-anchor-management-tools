## Auth Model

- **Middleware**: `src/middleware.ts` exists (Supabase SSR session refresh + short-link host routing, with a `PUBLIC_PATH_PREFIXES` allowlist and matcher `/((?!_next/static|_next/image|favicon.ico).*)`). A historical disabled copy `src/middleware.ts.disabled` remains in the tree from a prior Vercel incident.
- **Primary guard**: enforced in `src/app/(authenticated)/layout.tsx` via `createClient()` → `supabase.auth.getUser()`. Unauthenticated users are redirected to `/auth/login`. Users with zero management permissions (staff-portal-only roles such as `portal_shift_manager`) are redirected to `/portal/shifts`.
- **Permissions (RBAC)**: loaded server-side via `getUserPermissions()` ([[server-actions#rbac.ts|rbac.ts]]) and seeded into a client `PermissionContext`. Server actions re-check with `checkUserPermission(module, action, userId)`. Modules and actions are defined in `src/types/rbac.ts`. See [[relationships#Auth & RBAC]].
- **Cron auth**: cron routes call `authorizeCronRequest()` from `@/lib/cron-auth` (validates the Vercel Cron `Authorization: Bearer <CRON_SECRET>` header; `CRON_SECRET` required in production).
- **Public API auth**: external/website endpoints wrap handlers in `withApiAuth()` from `@/lib/api/auth` (API-key validation + rate limiting).
- **Webhook auth**: PayPal webhooks verify the provider signature; the Twilio webhook validates the Twilio signature (toggle via `SKIP_TWILIO_SIGNATURE_VALIDATION`).

Full detail in [[relationships#Auth & RBAC]].

## Layering

```
Pages / Client Components  →  Server Actions ('use server')  →  Services (src/services)  →  lib/ wrappers  →  Supabase / external SDKs
                              ↑ auth + RBAC + audit logging
API routes / Cron / Webhooks → Services / lib (own auth: API key, CRON_SECRET, signature)
```

- Mutations flow through `'use server'` actions ([[server-actions]]) that validate input, check RBAC, call services, write audit logs (`logAuditEvent`), and `revalidatePath`.
- Business logic lives in `src/services/` (~39 modules) and `src/lib/` (cross-cutting utilities & integration wrappers).
- Two Supabase clients: cookie/anon (`src/lib/supabase/server.ts`, respects RLS) and service-role admin (`src/lib/supabase/admin.ts`, system/cron only).
