## Auth Model

`src/middleware.ts` is present and active. It handles public path allow-listing and short-link host passthrough. Auth enforcement is delegated to layout after a prior Vercel incident.

Auth flow:
1. `middleware.ts` — allow-lists public prefixes (`/auth`, `/timeclock`, `/g`, `/r`, `/api`, etc.); sets `X-Robots-Tag: noindex` on protected pages
2. `(authenticated)/layout.tsx` — calls `supabase.auth.getUser()`; redirects unauthenticated users to `/auth/login`
3. `getUserPermissions()` RPC — fetches RBAC permission list for the user
4. Employees with zero management permissions are redirected to `/portal/shifts`
5. Per-page `checkUserPermission(module, action)` — granular access control
6. Server actions — re-verify auth and permissions server-side

RBAC roles: `super_admin`, `manager`, `staff`. Modules: `calendar`, `customers`, `employees`, `events`, `invoices`, `messages`, `parking`, `private-bookings`, `receipts`, `rota`, `leave`, `timeclock`, `payroll`, `settings`, `roles`.
