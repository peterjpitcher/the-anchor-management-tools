# Claim Summary PDF Redesign

**Date**: 2026-04-06
**Status**: Approved

## Overview

Redesign the quarterly Claim Summary PDF to fix formatting issues and add transaction detail pages. The PDF is generated by `src/lib/receipts/export/claim-summary-pdf.ts` using pdfkit and included in the quarterly receipts export bundle from `/api/receipts/export`.

## Problems with Current Implementation

1. **`addRow` layout bug** — uses `doc.y - doc.currentLineHeight()` to position the value on the same line as the label, which breaks when font sizes change between sections. Values misalign.
2. **MGD informational text truncation** — the "is not included in the claim total" note clips at the right margin due to insufficient width.
3. **No transaction details** — the PDF only shows aggregate totals with no supporting transaction data.

## Design

### Page 1: Summary (Cover Sheet)

Same sections as today with formatting fixes:

- **Header**: "The Anchor, Stanwell Moor Village" / "Quarterly Financial Claims Summary" / "Q4 2025 (October — December)" / Generated timestamp
- **Section 1: Mileage** — Total trips, Total miles, Miles @ £0.45, Miles @ £0.25, Tax year cumulative miles, **Mileage claim total** (bold)
- **Section 2: Expenses (Petty Cash)** — Total entries, Gross total, VAT reclaimable, **Expenses claim total** (bold)
- **Section 3: Machine Games Duty (Informational)** — Period, Collections in period, Total net takings, MGD due (20%), VAT on supplier. Note text: "For information only — MGD is paid directly to HMRC and is not included in the claim total."
- **Divider line**
- **Amount to Transfer to Owner** — Mileage amount, Expenses amount, **Grand Total** (bold, large). Only Mileage + Expenses contribute to the grand total. MGD is excluded.
- **Payment reference**: "Mileage Expenses {year} Q{quarter}" — e.g. "Mileage Expenses 2025 Q4". Displayed prominently below the grand total for use as a bank transfer reference.
- **Supporting documents list** — bullet list of files in the bundle

### Formatting Fix: `addRow` Replacement

Replace the current `addRow` function (which uses `doc.y - doc.currentLineHeight()` hack) with a proper two-column layout:

- Label positioned at x=50, value at x=400 (right-aligned to x=545)
- Both rendered on the same y coordinate
- Explicit y tracking — after rendering a row, advance y by the line height
- Bold variant sets font before rendering and resets after

### Formatting Fix: MGD Note Width

Increase the width parameter for the MGD informational text from 450 to the full available width (495 = 545 - 50 margin).

### Page 2+: Mileage Detail

- **Heading**: "Mileage — Trip Detail"
- **Table columns**: Date | Route | Miles | Amount (£)
- Data sorted by date ascending
- **Total row** at bottom (bold): sum of Miles, sum of Amount
- Auto-paginates via pdfkit's built-in page break when content exceeds page height

### Next page(s): Expenses Detail

- **Table columns**: Date | Company | Justification | Amount (£) | VAT (£)

- **Table columns**: Date | Net Take (£) | MGD 20% (£) | VAT on Supplier (£)

- Auto-paginates

### Data Flow

Currently `appendClaimSummaryPdf` receives summary objects (`MileageSummary`, `ExpensesSummary`, `MgdSummary`) but not the raw transaction data needed for detail tables.

**Change**: Extend `buildMileageCsv`, `buildExpensesCsv`, and `buildMgdCsv` return types to include the raw row arrays alongside the existing `csv` and `summary` fields. Then pass these arrays through to `appendClaimSummaryPdf` via an extended `ClaimSummaryInput` interface.

Specifically:
- `buildMileageCsv` already has `MileageTripRow[]` — add `rows: MileageTripRow[]` to return
- `buildExpensesCsv` already has `ExpenseRow[]` — add `rows: ExpenseRow[]` to return
- `buildMgdCsv` already has `MgdCollectionRow[]` — add `rows: MgdCollectionRow[]` to return

The row interfaces need to be exported from each CSV module so the PDF module can use them.

### Table Rendering

Implement a reusable `drawTable` helper within `claim-summary-pdf.ts` that:
- Accepts column definitions (label, width, alignment)
- Renders a header row with background fill and bold text
- Renders data rows with alternating or consistent styling
- Renders a bold total row at the end
- Handles page breaks by checking remaining page height before each row and calling `doc.addPage()` when needed, re-drawing the table header on the new page
