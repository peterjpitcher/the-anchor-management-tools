## Overview

Microsoft Graph is used to send application-level emails through a Microsoft 365 / Outlook mailbox. This covers invoices, quotes, and rota notifications. It is distinct from Supabase's built-in email system, which handles auth flows independently.

> [!NOTE] Two email systems in this project
> **Microsoft Graph** handles application emails (invoices, quotes, rota). **Supabase Auth** handles authentication emails (invites, password resets). Do not confuse the two — they are entirely separate pipelines.

## Environment Variables

| Variable | Purpose |
|---|---|
| `MICROSOFT_CLIENT_ID` | Azure AD application (client) ID |
| `MICROSOFT_CLIENT_SECRET` | Azure AD client secret |
| `MICROSOFT_TENANT_ID` | Azure AD tenant ID for the Microsoft 365 organisation |
| `MICROSOFT_USER_EMAIL` | The mailbox address emails are sent from |

## Library Files

| File | Purpose |
|---|---|
| `src/lib/microsoft-graph.ts` | Low-level Graph API client (token acquisition, HTTP requests) |
| `src/lib/email/emailService.ts` | Application-level `sendEmail()` function called by all modules |

## Authentication Method

The integration uses the **Azure AD client credentials flow** (app-only authentication). No user sign-in is required. The application authenticates directly with Azure AD using its client ID and secret, then calls the Graph API on behalf of the configured mailbox.

This means:
- No OAuth consent flow needed at runtime
- Token is cached and refreshed automatically
- The sending mailbox is fixed to `MICROSOFT_USER_EMAIL`

## Main Function

```typescript
sendEmail(
  to: string | string[],
  subject: string,
  html: string,
  cc?: string | string[],
  attachments?: EmailAttachment[]
): Promise<void>
```

All modules call `sendEmail()` — never the Graph API directly.

## PDF Attachments

PDF documents (invoices, quotes) are generated by Puppeteer via `src/lib/pdf-generator.ts`. The resulting buffer is passed as an attachment to `sendEmail()`, which encodes it as a base64 MIME attachment in the Graph API request.

> [!TIP] Attachment size
> Graph API supports attachments up to 3 MB via the standard messages endpoint. Larger attachments require the uploadSession endpoint. Current invoice and quote PDFs are well within the 3 MB limit.

## Rota Emails (Cron-driven)

Two rota emails are sent automatically each week via [[Cron Jobs]]:

| Email | Trigger | Recipient |
|---|---|---|
| Manager summary | Sunday 18:00 UTC | Managers |
| Staff schedule | Sunday 21:00 UTC | All scheduled staff |

These are triggered by Vercel cron routes and call `sendEmail()` directly — no user interaction required.

> [!WARNING] Cron email failures
> If Graph API authentication fails (e.g. expired client secret), cron emails will silently fail. Monitor the Vercel function logs for `microsoft-graph` errors around the Sunday cron windows.
